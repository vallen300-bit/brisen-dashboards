⭐ Proposed: Three-Tier Memory Architecture
★
MEMORY TIERS — Keeping Baker Sharp as Data Grows
Nothing is ever deleted. Data moves through tiers: full detail → compressed summaries → institutional knowledge. Retrieval prioritizes Tier 1, falls back to Tier 2/3 when needed.
Tier 1: Active Memory
0 – 90 days
• Full detail, every record
• Full vector embeddings (Voyage AI 1024d)
• Priority in retrieval
• All sources: email, WA, meetings, tasks, ClickUp
• Baker's "working desk" — nothing filtered
▶
Tier 2: Compressed (Opus)
90 days – 1 year
• Claude Opus compresses per-matter
• Preserves: figures, dates, commitments, dynamics
• Raw data → archive (never deleted)
• New summary vectors replace raw
• Cost: ~$12/month (30 matters × $0.10)
• LIVE — weekly Sundays 04:00 UTC
▶
Tier 3: Institutional (Sonnet)
1 year+
• Claude Sonnet distills to briefs
• "What happened & what was decided"
• Permanent strategic knowledge
• Cost: ~$2/month
• LIVE — monthly 1st of month 04:30 UTC
• Always recoverable from archive
⭐ Proposed: Slack Integration — Closing the Learning Gap
★
SLACK MEMORY — Today's Messages Disappear. Tomorrow They Won't.
Currently, Slack messages Baker composes via Cowork are generated once and forgotten. With this integration, they become part of Baker's permanent learning.
Inbound (team → Baker)
• Channel messages (all public)
• DMs to Baker
• Thread replies & reactions
• → slack_messages table
• → baker-slack vectors
Outbound (Baker → team)
• Messages Baker composes via Cowork
• Stored WITH context (who, channel, topic)
• Baker learns his own tone & style
• Future messages reference past patterns
• Builds communication memory
Tier Lifecycle for Slack
• 0–90d: Every message searchable
• 90d–1yr: Weekly channel summaries
• 1yr+: Monthly communication digest
• High-value messages stay in Tier 1
• Est. growth: ~200–500 msgs/day

World → Baker ingests → Baker remembers
All channels feed raw data into Sentinel's memory stores (PostgreSQL + Qdrant)
2

14 read tools surface deadlines, emails, contacts, tasks, meetings, conversations
