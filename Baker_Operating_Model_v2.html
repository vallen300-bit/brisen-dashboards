Task Flow — One Relay Hop
1
PM writes brief
TASK / WHAT / WHY / ACCEPT — 2-3 sentences each. No "how."
2
Director relays
Copy-pastes to Code terminal. One hop, no rewriting.
3
Code ships
Plan mode → Director approves → implement, test, push. Reports: commit hash, files, what to test.
4
Director confirms
Tells PM "Code shipped [X]." Code may also post to ClickUp Handoff Notes.
5
PM verifies
Checks ClickUp Handoff Notes + GitHub commits. Verifies acceptance criteria. Updates state.

Session Protocols

PM Sessions

On Open
1
Read PM_ONBOARD.md (~73 lines) — only startup read
2
80-LINE CEILING: If over 80 lines, trim first. Hard rule.
3
Connect to ClickUp — check Handoff Notes for Code replies
4
If Code shipped, review commits on GitHub
5
Pick up from pending list
On Close
1
Update PM_ONBOARD.md — delete completed, add state, keep <80 lines
2
Append session summary to PM_HANDOVER_PROMPT.md
3
Save new briefs to pm/briefs/
