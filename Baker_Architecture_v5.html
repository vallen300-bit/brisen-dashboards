Baker System Architecture

Three actors, three jobs — one memory
v5.1 — March 2026 — 18 MCP tools live
The Three Actors
🛡️

Baker (Sentinel)

Chief of Staff
"The assistant who never sleeps"
✓ What it does Monitors all channels — email, WhatsApp, meetings, ClickUp, RSS. Stores everything in memory. Acts on routine tasks: draft emails, update ClickUp, surface deadlines. Answers questions via Scan. Always on guard.
✗ What it doesn't do Think deeply. Analyze documents. Brainstorm. Create strategy. Make decisions.
🧠

Cowork + MCP

The Thinker & Creator
"The working session at the Director's desk"
✓ What it does Deep analysis of legal docs, financial tables, negotiations. Cross-references Baker's memory with new material. Brainstorms solutions. Prepares decisions. Creates documents.
✗ What it doesn't do Monitor channels. Run 24/7. Act autonomously.
👔

Director (Dimitry)

Final Authority
"The Chairman"
✓ What it does Decides. Approves. Directs both Baker and Cowork.
✗ What it doesn't do Everything else — that's what Baker and Cowork are for.
How They Connect
From To Via Direction What Flows
World Baker Triggers — Gmail, WhatsApp, Fireflies, ClickUp, RSS, Whoop, Dropbox → Inbound Raw data: emails, messages, transcripts, tasks, articles, health
Baker Baker Pipeline — Classify → Retrieve → Augment → Generate → Store ↻ Internal Processed intelligence: embeddings, structured records, briefings
Baker Director CEO Cockpit (Scan) + WhatsApp → Outbound Briefings, answers, alerts, draft emails
Director Baker Scan chat + WhatsApp ← Inbound Questions, approvals, commands
Baker Cowork MCP Server — 14 read tools + raw SQL → Read Memory: deadlines, emails, contacts, tasks, meetings, conversations, health, RSS, analyses
Cowork Baker MCP Server — 4 write tools (live) ← Write-back Decisions, deadlines, VIP contacts, analysis reports
Cowork Director Session output — handover files, documents → Outbound Analysis, strategies, prepared decisions
The Memory Cycle
1
World → Baker ingests → Baker remembers
All channels feed raw data into Sentinel's memory stores (PostgreSQL + Qdrant)
2
Director opens Cowork → MCP reads Baker's memory
14 read tools surface deadlines, emails, contacts, tasks, meetings, conversations
3
Director + Cowork think together → reach decision
Deep analysis, cross-referencing, brainstorming — the working session
4
Cowork writes back to Baker via MCP
Decisions, notes, analysis conclusions flow back into Baker's memory
5
Baker remembers the decision
Stored in structured DB + vector memory — available to all future queries
6
Baker acts on it → World
Next trigger cycle picks up the decision — emails sent, tasks updated, alerts fired
"Baker never forgets, Cowork never stops thinking.
The MCP bridge makes them one system."
System Design Principle
The MCP Bridge — 18 Tools
The direct connection between Cowork (thinking sessions) and Baker (24/7 memory). Cowork reads everything Baker knows and writes back what the Director decides.

Read Tools

14 TOOLS

baker_briefing_queueItems queued for next daily briefing

4 TOOLS
